第03章 需求分析概要

辽宁师范大学 • 张大为@https://daweizh.github.io/jw2/

对于AMC、TAM和SAM项目，我们大概的项目定位是：

  1. AMC项目管理用户、应用、角色，应用与角色之间的权限及用户与角色之间的关系。
  2. TAM项目管理的是某老师可能的各项成果，如：发表的论文、获批的项目、出版的书籍、获得的奖项和授权的专利等。
  3. SAM项目管理的是某同学可能的各项成果，如：竞赛的获奖、参与的项目、获得的奖励、发表的文章和授权的专利等。

3.1 应用管理中心(AMC)

基本功能可能包括：

  1. 系统账户和用户的管理，角色和功能的管理。不同角色用户（包括无用户）使用不同功能，不同级别角色可以实现继承。
  2. 定制系统功能的挂载和发布，基于用户和角色定制的功能发布。
  3. 权限及安全保障

可能涉及到数据表格包括：

  1. 用户(user)表:账号，密码，电邮，角色。
  2. 个人信息(profile):账号，电话，头像......
  3. 角色(role)：id,name
  4. 角色权限(premission):id,roid，功能串....
  5. 功能菜单(menu):id,.....

上述5个表系统管理员可以增、删、改、查。用户表和个人信息表，用户登录后可以修改属于自己的一条信息。用户采用唯一的账户和密码登录，根据账户的权限分配不同的角色，产生不同的界面功能。

典型的应用场景及相关的权限和功能变化如下例：

  1. 辛守强八年前是某大学的本科生，正准备报考辽宁师范大学的硕士研究生，经常浏览关注本系统发布的各种信息。
  2. 辛守强六年前已经成为学院研究生生学生会主席，承担就业外联工作，负责组织修改毕业同学的就业建议（简历审阅，批注和投放指导）。
  3. 辛守强三前年博五，采用研究生账户登录完成毕业生相关工作，涉及到的功能包括（提交论文，毕业信息填报）。
  4. 辛守强去年毕业，成为计算机与信息技术学院的老师，现在需要考虑分配教师账号并分配教师相关的功能（填写教学计划，提交教学日历，填报科研成果）。
  5. 辛守强今年因业务能力超强成为学院科研副院长，需要增加权限（科研成果审定，科研成果统计，报表）。
  6. 辛守强工作若干年后因工作调整，需要离开教室工作岗位，信息系统也应做出相应的用户权限修订。

管理信息系统基础功能，应能很好的满足上述辛守强在工作中遇到的各种权限和功能的调整，并最终体现在辛守强登录后的系统界面和工作流程上。

3.2 教师成果管理(TAM)

教师登录后选择论文、专著、项目、专利或获奖之一进行管理。管理内容包括该项成果的列表查询，导出到excel，表内修改，新增成果，修改成果，删除成果等。每一项成果需要上传一到多个附件支撑。每一项成果需要考虑排序和所属单位，与合作者的关系等。

可能涉及到数据表格包括：

  1. 论文(paper)表:id,uid,title,...。
  2. 专利(patent)表:id，uid，title,no,......
  3. 项目(project)表：id,uid,title,no,.....
  4. 著书(book)表:id,uid,title,isbn,....
  5. 获奖(award)表:id,uid,title,no,.....
  6. ... ...

3.3 学生成果管理(SAM)

学生登录后选择论文、竞赛、项目、专利或获奖之一进行管理。管理内容包括该项成果的列表查询，导出到excel，表内修改，新增成果，修改成果，删除成果等。每一项成果需要上传一到多个附件支撑。每一项成果需要考虑排序和所属单位，与合作者的关系等。

可能涉及到数据表格包括：

  1. 论文(paper)表:id,uid,title,.....
  2. 专利(patent)表:id,uid,title,no,......
  3. 项目(project)表：id,uid,title,no,....
  4. 竞赛(contest)表:id,uid,title,no,......
  5. 获奖(award)表:id,uid,title,no,.....
  6. ... ...
